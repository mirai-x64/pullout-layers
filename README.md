# 引き抜き層 — pullout-layers

落ちてくる方には触れない落ち物パズル。`index.html` をダブルクリックで動く(依存なし)。

落下中の層は操作できない。できるのは**最下段 h₀ を1つ引き抜くこと**だけで、
引き抜くとその列が1段沈む。結合は引き抜きでのみ起きる。同色が縦横3つ以上で消え、
上が沈んで連鎖する。いずれかの列が天井に達すると終了。

題材は T²MLR の中間層に残る隠れ状態。積まれた山が推論の層、最下段が最古の状態で、
各マスの層番号ラベル `h₀ h₁ h₂ …` と結合時の発光だけで表現している。

## 数値の根拠

`tools/balance.mjs` が反応遅れつきの自動操縦を3種類走らせ、実時間を待たずに掃引する。

| 操縦 | 見る範囲 | 生存秒 | 死亡率 |
|---|---|---|---|
| 無作為 | 引き抜ける列から適当に | 53s | 8/8 |
| 人間相当 | 目についた4列だけ評価 | 140s | 7/8 |
| 全列読み | 8列すべてを完全評価 | 190s | 4/8 |

「どのマスを引き抜くか」に意味があることは、この3者の差が示している。
差が無ければ選択は飾りでしかない。

## 掃引で分かったこと

**落下間隔を詰めても山は育たない。** 出現速度を 0.34s → 0.16s と倍以上にしても
平均の高さは 2.35 → 1.84 と逆に下がり、盤面が平らなまま急死するだけだった。

理由は負のフィードバックにある。消去は盤面が平らで広いときほど作りやすい
(最下段8マスの横3つ揃いが偶然できる)ため、山が育つ端から刈り取られ、
均衡が床に張り付く。出現を速めても均衡の高さは上がらず、寿命が縮むだけになる。

効いた手は2つ。

- **色数を 4 → 6 に増やす。** 4色だと最下段の横3つが偶然できすぎる。
  消去を意図的なものにして初めて山が積める。
- **立ち上がりを 70s → 30s に縮める。** 最初の1分が無風だと、
  緊張が来る前にゲームが終わる。平均の高さもこの助走に薄められていた。

なお**引き抜きそれ自体が質量を消す**という構造は残っている。クールダウン0.6秒は
無条件に 1.67 マス/秒 の除去にあたり、出現の相当部分を消去に頼らず吸収してしまう。
山をもっと高く保ちたければ、次に触るのはこの一手 — 引き抜いたマスを消さずに
どこかへ移す、あたりになる。

## 道具

- `node tools/balance.mjs` — 均衡の掃引。`CONFIGS` / `TRIALS` / `CAP` / `DELAY` を環境変数で渡す
- `node tools/playtest.mjs [秒数]` — 実描画・実クリックでの確認。
  balance.mjs は `step()` だけを回して `draw()` を一度も通らないので、
  描画とヒットテストはこちらでしか確かめられない
